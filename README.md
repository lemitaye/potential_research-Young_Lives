# Mother Tongue Instruction, Learning and Employment: Evidence from Ethiopia

Replication code and manuscript for the paper by **Teferi Mergo** (University of
Waterloo) and **Lemi Daba** (SPIA CGIAR). The study estimates the causal effect
of mother-tongue instruction (MTI) on cognitive test scores and employment using
the **Young Lives** longitudinal survey of Ethiopia (rounds 1–5, 2002–2016) plus
the 2012–13 School Survey, via an **IV / 2SLS** design.

## Reproducing the results

**Requirements:** R (≥ 4.x) and a LaTeX distribution. R packages:
`tidyverse`, `haven`, `scales`, `stargazer`, `AER`, `lfe`, `fastDummies`,
`xtable`, and `starpolishr` (GitHub only):

```r
install.packages(c("tidyverse","haven","scales","stargazer","AER","lfe","fastDummies","xtable"))
remotes::install_github("ChandlerLutz/starpolishr")
```

**Run the whole pipeline** from the project root:

```r
source("scripts/master.R")     # or:  Rscript scripts/master.R
```

Generated tables/figures are written to `output/`. Review them and copy the ones
the manuscript uses into `MTI__learning_and_job_outcomes/tables|figures`, then
compile `main.tex` (`table_rev.R` already writes Table 3 straight into the
manuscript). See the mapping and caveats below.

### Pipeline (order matters)

| # | Script | Produces |
|---|--------|----------|
| 1 | `scripts/extract_clean.R` | `data/aa_samp.csv`, `data/non_aa_samp.csv` (older cohort) |
| 2 | `scripts/cleaning_yc.R` | `data/ylsample_yc.csv`, `data/schsur_yl_joined.csv` (younger cohort) |
| 3 | `scripts/descriptives.R` | `output/tables/tableI.tex`, `output/figures/employ_gr.pdf` |
| 4 | `scripts/estimation.R` | fits all regression models into the R workspace |
| 5 | `scripts/export_res.R` | `output/tables/tableII…VI.tex` — **consumes step-4 models in memory** |
| 6 | `scripts/table_rev.R` | `MTI__learning_and_job_outcomes/tables/table_rev.tex` (Table 3) |

> **Workspace gotcha:** `descriptives.R` and `estimation.R` each start with
> `rm(list = ls())` (each reloads its own data). `export_res.R` has **no**
> `rm()` and depends on the models `estimation.R` leaves in memory, so it must
> run in the same session immediately after it. `master.R` handles this order.

## Manuscript table/figure → source

Table **numbers** follow the `\input` order in `main.tex`, which does **not**
match the file names.

| In paper | File (`…/tables/`) | Generated by | Reproduction status |
|----------|--------------------|--------------|---------------------|
| Table 1 | `tableI.tex` | `descriptives.R` (xtable) | reproducible |
| Table 2 | `tableII_mod.tex` | `export_res.R` (see caveat) | ⚠ earlier vintage on disk |
| Table 3 | `table_rev.tex` | `scripts/table_rev.R` | ✅ **byte-verified** against manuscript |
| Table 4 | `tableIII.tex` | `export_res.R` (see caveat) | ⚠ earlier vintage on disk |
| Table 5 | `tableV.tex` | `export_res.R` (see caveat) | ⚠ earlier vintage on disk |
| Figure 1 | `figures/employ_gr.pdf` | `descriptives.R` (ggsave) | reproducible |
| Appendix map | `figures/map.png` | static image (not generated) | n/a |

Files present but **commented out** in `main.tex` (not currently in the paper):
`tableII.tex`, `tableIV.tex` (younger cohort), `tableVI.tex` (excluding Amhara).

### ⚠ Reproduction caveat (Tables 2, 4, 5)

The manuscript's current Table 2 (`tableII_mod`), Table 4 (`tableIII`) and
Table 5 (`tableV`) were generated together on **3 Dec 2023** on a Windows machine
(the `%Tex File: E:\…` header in each). The `export_res.R` in this repo is an
earlier **Nov 2022** vintage and produces different, earlier versions (e.g. it
writes `tableII.tex`, not the restructured `tableII_mod.tex`). To reproduce those
three tables **exactly**, the Dec-2023 generator must be recovered from the
author's machine or reconstructed — exactly as was done for
`scripts/table_rev.R`, which now reproduces Table 3 to the digit.
